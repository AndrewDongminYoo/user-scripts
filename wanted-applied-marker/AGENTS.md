# AGENTS.md (wanted-applied-marker)

Subdirectory guide for `wanted-applied-marker/`.

## Purpose

- TypeScript + Vite userscript package for the Wanted applied-job marker.
- Builds `dist/wanted-applied-marker.user.js` via `vite-plugin-monkey`.

## Current Layout

```text
wanted-applied-marker/
├── package.json
├── tsconfig.json
├── vite.config.ts   # monkey plugin: metadata header, match, grants, version
├── README.md
├── dist/            # build output (gitignored)
├── test/
│   └── run.mjs      # Node vm-sandbox harness against the built dist bundle
└── src/
    ├── main.ts      # userscript logic (import-free, GM_* at runtime)
    └── env.d.ts     # GM_getValue/GM_setValue global declarations
```

## What To Change Here

- Userscript behavior: edit `src/main.ts`.
- Metadata (match globs, grants, name, version fallback): edit `vite.config.ts`.
- Regression coverage: extend `test/run.mjs`; run `pnpm build` before `pnpm test`.
- GM API surface: extend the `declare global` block in `src/env.d.ts`.
- Keep docs in `docs/plans/` aligned with actual progress after each milestone.

## Commands

Run inside this directory:

```bash
pnpm install
pnpm dev
pnpm build
pnpm preview
pnpm test
```

## Constraints

- Keep TypeScript strict mode intact unless there is a documented reason (`tsconfig.json` sets `strict: true`).
- Keep `src/main.ts` import-free; `vite-plugin-monkey` supplies the metadata header and GM globals.
- Do not hand-edit `dist/*.user.js`; it is regenerated by `pnpm build`.
- Keep `/wdlist*` metadata broad enough for both supported hosts, and preserve the exact runtime pathname guard that rejects unrelated prefixes.
