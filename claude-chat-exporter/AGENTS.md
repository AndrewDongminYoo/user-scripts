## Design Notes

- **Auth:** rides the in-page authenticated session with same-origin credentialed
  `fetch`; never forge or reconstruct tokens. Org id comes from the `lastActiveOrg`
  cookie with a `GET /api/organizations` fallback.
- **API seam:** `getOrgId` / `fetchConversation` / `toMarkdown` are isolated so a
  Claude-side API change is a local repair, not a rewrite.
- **Grants:** `@grant GM_addStyle, GM_getValue, GM_setValue`. `GM_addStyle` is
  required for CSP: claude.ai ships a strict CSP (`script-src 'nonce-‚Ä¶'
'strict-dynamic'`, no `'unsafe-inline'`), so a `@grant none` script ‚Äî which
  Tampermonkey injects into the page's main world without a nonce ‚Äî is blocked by
  CSP and never runs (no button appears). Declaring a real GM\_\* grant makes
  Tampermonkey run the script in its sandboxed world, which is exempt from the page
  CSP. `GM_addStyle` doubles as the button's CSS injector. The file download uses a
  `Blob` + temporary anchor, which works in the sandbox.
- **Settings:** the ‚öôÔ∏è panel persists `{ format, frontmatter, messageTimestamps, includeThinking, includeToolCalls, includeAttachments }` under the `cce_settings` key via `GM_getValue`/`GM_setValue`.
- **Rich-content rendering seam:** two independent passes read the same `content[]` blocks.
  `renderBlocks` builds the Markdown body ‚Äî document-order `<details>` sections, each truncated to `MD_BLOCK_CAP` (2000 chars).
  `collectStructured` builds the JSON body ‚Äî typed `thinking[]` / `tools[]` / `attachments[]` arrays, untruncated.
  Keep both in sync when the block shape changes; they intentionally duplicate the block-walk instead of sharing one code path so Markdown truncation never leaks into JSON.
- **Tool pairing:** `tool_use`/`tool_result` blocks are paired by document order (a `pending` index tracks the last unmatched `tool_use`), not by an id map ‚Äî verified 1:1 against live API responses.
  A `tool_result` with no preceding `tool_use` becomes its own record instead of being dropped.
- **Deliberately not exported:** uploaded image attachments (`files[]`) and text-block `citations` are out of scope ‚Äî only `attachments[].extracted_content` (text extracted server-side) and `content[]` blocks are read.
- **Scope:** exports the operator's own conversations only. No detection evasion,
  no mass collection, polite single-request fetches.

## Constraints

- Keep TypeScript strict mode intact (`tsconfig.json` sets `strict: true`).
- Keep `src/main.ts` import-free; `vite-plugin-monkey` supplies the metadata header.
- Do not hand-edit `dist/*.user.js`; it is regenerated by `pnpm build`.
