# AGENTS.md (claude-chat-exporter)

Subdirectory guide for `claude-chat-exporter/`.

## Purpose

- TypeScript + Vite userscript that exports Claude.ai conversations to Markdown.
- Builds `dist/claude-chat-exporter.user.js` via `vite-plugin-monkey`.

## Current Layout

```text
claude-chat-exporter/
├── package.json
├── tsconfig.json
├── vite.config.ts   # monkey plugin: metadata header, match, run-at, grant none
├── README.md
├── dist/            # build output (gitignored)
└── src/
    ├── main.ts      # UI buttons + Claude API fetch + Markdown + store-only ZIP
    └── env.d.ts     # GM_addStyle global (see Grant note below)
```

## What To Change Here

- Export behavior / Markdown format: edit `src/main.ts`.
- Metadata (match globs, run-at, name, version fallback): edit `vite.config.ts`.

## Commands

Run inside this directory:

```bash
pnpm install
pnpm dev
pnpm build
pnpm preview
```

## Design Notes

- **Auth:** rides the in-page authenticated session with same-origin credentialed
  `fetch`; never forge or reconstruct tokens. Org id comes from the `lastActiveOrg`
  cookie with a `GET /api/organizations` fallback.
- **API seam:** `getOrgId` / `fetchConversation` / `toMarkdown` are isolated so a
  Claude-side API change is a local repair, not a rewrite.
- **Grant:** `@grant GM_addStyle` (required). claude.ai ships a strict CSP
  (`script-src 'nonce-…' 'strict-dynamic'`, no `'unsafe-inline'`), so a `@grant none`
  script — which Tampermonkey injects into the page's main world without a nonce — is
  blocked by CSP and never runs (no button appears). Declaring a real GM\_\* grant makes
  Tampermonkey run the script in its sandboxed world, which is exempt from the page
  CSP. `GM_addStyle` doubles as the button's CSS injector. The file download still uses
  a `Blob` + temporary anchor, which works in the sandbox.
- **Scope:** exports the operator's own conversations only. No detection evasion,
  no mass collection, polite single-request fetches.

## Constraints

- Keep TypeScript strict mode intact (`tsconfig.json` sets `strict: true`).
- Keep `src/main.ts` import-free; `vite-plugin-monkey` supplies the metadata header.
- Do not hand-edit `dist/*.user.js`; it is regenerated by `pnpm build`.
